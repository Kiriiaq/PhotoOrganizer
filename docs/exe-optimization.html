3. Findings détaillés

[F-01] ExifTool Perl (10.31 MB) — fallback non opérationnel FORT

Gain estimé : ~10.3 MB compressé · Effort : Faible · Risque : Moyen

ExifTool est invoqué comme fallback subprocess (src/core/metadata/exif_extractor.py:299). Les méthodes primaires (exifread, Pillow, pillow_heif) couvrent l'usage cible (JPEG/PNG/HEIC/RAW/vidéos). Le code cherche le binaire à assets/exiftool.exe alors que le bundle le place à assets/tools/exiftool.exe — donc le fallback est cassé en pratique (sauf si exiftool est dans le PATH système).

Avant — build.py:176-179 
for data_dir in ["assets", "resources", "src"]:
    src_path = project_dir / data_dir
    if src_path.exists():
        cmd.extend([
            "--add-data",
            f"{src_path}{os.pathsep}{data_dir}"
        ])
Après — whitelist assets/icons 
ASSETS_INCLUDE = ["icons"]  # tools/ exclu
for sub in ASSETS_INCLUDE:
    src_path = project_dir / "assets" / sub
    if src_path.exists():
        cmd.extend([
            "--add-data",
            f"{src_path}{os.pathsep}assets/{sub}"
        ])
for data_dir in ["resources", "src"]:
    src_path = project_dir / data_dir
    if src_path.exists():
        cmd.extend([
            "--add-data",
            f"{src_path}{os.pathsep}{data_dir}"
        ])

Risque : Perte du fallback pour fichiers exotiques (DNG anciens, MOV propriétaires). Atténuation : documenter dans le README qu'ExifTool peut être installé via winget install exiftool, le code de détection (_find_exiftool) le retrouvera dans le PATH.

[F-02] Build depuis un venv minimal isolé (CRITIQUE) FORT

Gain estimé : ~5–6 MB · Effort : 5 min · Risque : Nul

Le Python global utilisé pour le build contient 325 paquets (numpy 2.4.4, pandas 3.0.2, scipy 1.17.1, matplotlib 3.10.9, cryptography 48.0.0, etc.). PyInstaller collecte des hooks/datas inattendus même avec --exclude-module.

# Création
python -m venv .venv-build
.\.venv-build\Scripts\Activate.ps1

# Installation stricte
pip install --upgrade pip
pip install -r requirements.txt
pip install pyinstaller>=6.0

# Build
python build.py

Variante automatisée — ajouter un garde-fou en début de build.py :

def _require_clean_env():
    import importlib.util
    forbidden = ["numpy", "pandas", "scipy", "matplotlib", "jupyter", "ipython"]
    found = [m for m in forbidden if importlib.util.find_spec(m)]
    if found:
        print(f"[WARN] Build env pollué : {found}")
        print("       Utiliser un venv minimal : .venv-build")
        if "--allow-polluted-env" not in sys.argv:
            sys.exit(1)

[F-03] pillow-heif → pi-heif (sans libx265) FORT

Gain estimé : ~2.3 MB compressé (libx265-215-…dll) · Effort : 10 min · Risque : Faible

pillow_heif 1.3.0 embarque libx265 (encodeur HEVC) alors que le projet ne fait que lire des HEIC. Le sister-package pi-heif (même auteur) fournit l'API register_heif_opener() identique sans libx265.

Avant — requirements.txt 
pillow-heif>=0.13.0
Après 
pi-heif>=0.20.0

Aucun changement requis dans le code (from pillow_heif import register_heif_opener fonctionne avec pi-heif qui expose le même module top-level pour compatibilité — à confirmer sur la doc upstream avant d'appliquer).

Risque : Faible — l'app n'écrit jamais de HEIC.

[F-04] PIL._avif (1.79 MB) non requis FORT

Gain estimé : ~1.8 MB compressé · Effort : 5 min · Risque : Faible

Pillow 12+ embarque _avif.cp311-win_amd64.pyd (7.7 MB sur disque, 1.79 MB compressé). Le projet liste .avif comme extension scannée mais l'extraction métadonnées passe par exifread ou PIL.Image.open() ; les vraies photos AVIF sont rares dans l'usage cible (smartphone JPEG/HEIC).

Option A — pin Pillow < 12 (recommandé) 
# requirements.txt + pyproject.toml
Pillow>=10.0.0,<12.0.0
Option B — exclude module (plus fragile) 
# build.py
EXCLUDE_MODULES = [..., "PIL._avif"]

Risque : Si un utilisateur a des .avif, les métadonnées EXIF profondes ne seront pas lues, mais _extract_basic fournit toujours FileName/Size/MTime.

[F-05] icon.png sur-dimensionné (945 KB) MOYEN

Gain estimé : ~850 KB · Effort : Trivial · Risque : Nul

Un PNG d'icône devrait peser 30-100 KB max. La compression actuelle est défavorable.

# Avec pngquant (256 couleurs, qualité 80-95%)
pngquant --quality=80-95 --strip --skip-if-larger \
  --ext .png --force assets/icons/icon.png

# OU avec oxipng (lossless agressif)
oxipng -o max --strip safe assets/icons/icon.png

[F-08] requests pour 1 seul GET → urllib.request stdlib MOYEN

Gain : ~140 KB requests stack + libère cryptography indirectement · Effort : 30 min · Risque : Faible

Le seul appel HTTP du projet est un GET vers Nominatim (src/core/metadata/gps_processor.py:216). Trivialement remplaçable.

Avant 
import requests

url = "https://nominatim.openstreetmap.org/reverse"
params = {'format':'json','lat':lat,'lon':lon,
          'zoom':16,'addressdetails':1}
headers = {'User-Agent': 'PhotoOrganizer/2.0'}

response = requests.get(url, params=params,
                        headers=headers, timeout=5)
if response.status_code == 200:
    data = response.json()
Après — stdlib uniquement 
import json
import urllib.parse
import urllib.request

url = "https://nominatim.openstreetmap.org/reverse"
params = {'format':'json','lat':lat,'lon':lon,
          'zoom':16,'addressdetails':1}
full = f"{url}?{urllib.parse.urlencode(params)}"
req = urllib.request.Request(
    full, headers={'User-Agent':'PhotoOrganizer/2.0'})
try:
    with urllib.request.urlopen(req, timeout=5) as r:
        if r.status == 200:
            data = json.loads(r.read().decode('utf-8'))
except Exception as e:
    logger.debug(f"Géocodage échoué: {e}")
    data = None

Puis retirer requests / urllib3 / charset_normalizer / idna de HIDDEN_IMPORTS et de requirements.txt / pyproject.toml.

[F-09] cryptography (4.19 MB) embarqué inutilement FORT

Gain : ~4.2 MB compressé · Effort : 2 min (après F-02) · Risque : Faible

cryptography 48.0.0 est installé globalement (tiré par msoffcrypto-tool, pdfminer.six, pyAesCrypt — aucun n'est utilisé par PhotoOrganizer). PyInstaller le détecte via le runtime hook pyi_rth_cryptography_openssl.

# build.py
EXCLUDE_MODULES = [..., "cryptography", "cffi", "_cffi_backend"]

Risque : Vérifier qu'urllib.request HTTPS fonctionne sans cryptography (= utilise ssl stdlib, OK). Si on garde requests (F-08 non appliqué), urllib3 ≥ 2 retombe sur ssl stdlib si cryptography est absent.

[F-10] chardet (823 KB) tiré inutilement MOYEN

Gain : ~820 KB compressé · Effort : 2 min · Risque : Faible

Probablement tiré par requests (qui dépend de charset_normalizer OU chardet). Aucun import direct dans src/.

# build.py
EXCLUDE_MODULES = [..., "chardet"]

[F-11] Fichiers .DS_Store macOS embarqués FAIBLE

Gain : ~12 KB · Effort : 10 min en mode .spec. customtkinter/assets/.DS_Store + customtkinter/assets/icons/.DS_Store sont auto-collectés. Résolu par le hook _dedupe du .spec proposé en §5.

[F-12] Thèmes CustomTkinter inutilisés (dark-blue.json, green.json) FAIBLE

Gain : ~9 KB · L'app utilise set_default_color_theme("blue") en dur (src/ui/app.py:62). Effort > bénéfice — laisser tel quel.

[F-13] --onefile vs --onedir — décision humaine MOYEN

--onefile décompresse dans %TEMP%\_MEIxxxxx\ à chaque lancement (1–3 s sur HDD + scan Defender). --onedir = démarrage instantané, plus gros sur disque mais distributable en ZIP, et patch incrémental possible.

[F-14] UPX désactivé (--noupx) MOYEN

Gain : ~3–5 MB · Effort : 30 min · Risque : Moyen (faux-positifs Defender)

winget install upx-project.upx

# Dans build.py

    for excl in ["vcruntime140.dll", "python311.dll",

        cmd.extend(["--upx-exclude", excl])

[F-15] EXCLUDE_MODULES à enrichir FAIBLE

Gain : 200–500 KB. Ajouter cryptography, cffi, chardet (cf. F-09/F-10), encodings asiatiques rares (cp932, cp949, cp950, euc_jp, euc_kr, gb2312, gbk, iso2022_jp, iso2022_kr), email.mime, email.policy, xmlrpc, wsgiref.

[F-16] Duplications DLL (perl532.dll ×2, libstdc++-6.dll ×2) MOYEN

Gain : ~1 MB. perl532.dll est résolu par F-01 (retrait ExifTool). Les libstdc++-6.dll et libde265-0-…dll doublons proviennent d'un double-passage de PyInstaller sur les hooks et le binary collection. Résolu par le filtre _dedupe dans le .spec proposé.

[F-17] --optimize=2 RÉF

Déjà actif Bénéficie au PYZ (3.47 MB). Aucune action.

[F-18] --strip RÉF

Déjà actif Limité sous Windows (nécessite strip.exe de MinGW). Gain marginal ~100-300 KB si présent, sinon no-op silencieux.

[F-19] Migrer build.py → .spec versionné FAIBLE

Refactor recommandé pour bénéficier du filtrage post-Analysis (_dedupe, exclusions .DS_Store, etc.). Cf. .spec complet en §5.

5. .spec PyInstaller optimisé proposé

À placer en PhotoOrganizer.spec et invoquer via pyinstaller PhotoOrganizer.spec.

# -*- mode: python ; coding: utf-8 -*-
"""PhotoOrganizer build spec - optimise taille.

Prealable : creer un venv minimal isole (cf. F-02) :
    python -m venv .venv-build
    .\.venv-build\Scripts\Activate.ps1
    pip install -r requirements.txt pyinstaller>=6.0
"""
import os
from pathlib import Path

APP_NAME = "PhotoOrganizer"
VERSION = "2.0.0"
PROJECT_DIR = Path(SPECPATH)

HIDDEN_IMPORTS = [
    "customtkinter", "darkdetect",
    "PIL", "PIL._imaging", "PIL._imagingft",
    "exifread", "pillow_heif",   # ou pi_heif (cf. F-03)
    "sqlite3", "_sqlite3",
    "yaml",
    "tkinterdnd2", "plyer", "plyer.platforms.win.notification",
]

EXCLUDE_MODULES = [
    # Libs lourdes jamais utilisees
    "scipy", "cv2", "dlib", "moviepy", "whisper", "oletools",
    "pandas", "numpy", "openpyxl", "fitz", "pymupdf",
    "docx", "pptx", "PyPDF2", "reportlab", "matplotlib", "seaborn", "win32com",
    "IPython", "jupyter", "notebook", "sphinx",
    "tornado", "zmq", "babel",
    "PyQt5", "PyQt6", "PySide2", "PySide6", "wx",
    "pytz", "dateutil", "blake3",
    # F-09, F-10
    "cryptography", "cffi", "_cffi_backend", "chardet",
    # F-04 option B (fragile)
    # "PIL._avif",
    # F-08 (si applique)
    # "requests", "urllib3", "charset_normalizer", "idna", "certifi",
    # Stdlib outils dev
    "unittest", "test", "tests", "pytest", "pydoc", "doctest",
    "lib2to3", "ensurepip", "venv", "distutils",
    "setuptools", "pkg_resources", "pip",
    "tkinter.test", "idlelib",
    "asyncio.test_support", "concurrent.futures.process",
    "email.test", "xmlrpc", "wsgiref",
    "ruff", "mypy", "vulture", "bandit",
    # Encodings Asie peu utilises (F-15)
    "encodings.cp932", "encodings.cp949", "encodings.cp950",
    "encodings.euc_jp", "encodings.euc_kr",
    "encodings.gb2312", "encodings.gbk",
    "encodings.iso2022_jp", "encodings.iso2022_kr",
]

# Datas - F-01 : exclut assets/tools/
datas = []
icons_dir = PROJECT_DIR / "assets" / "icons"
if icons_dir.exists():
    datas.append((str(icons_dir), "assets/icons"))
res_dir = PROJECT_DIR / "resources"
if res_dir.exists():
    datas.append((str(res_dir), "resources"))
datas.append((str(PROJECT_DIR / "src"), "src"))

try:
    import tkinterdnd2 as _tkdnd
    tkdnd_dir = Path(_tkdnd.__path__[0]) / "tkdnd"
    if tkdnd_dir.is_dir():
        datas.append((str(tkdnd_dir), "tkinterdnd2/tkdnd"))
except ImportError:
    pass


a = Analysis(
    [str(PROJECT_DIR / "main.py")],
    pathex=[str(PROJECT_DIR)],
    binaries=[],
    datas=datas,
    hiddenimports=HIDDEN_IMPORTS,
    hookspath=[],
    hooksconfig={},
    runtime_hooks=[],
    excludes=EXCLUDE_MODULES,
    noarchive=False,
    optimize=2,  # F-17
)

# Post-Analysis cleanup : .DS_Store + dedupe dll (F-11, F-16)
_seen = set()
def _dedupe(files):
    out = []
    for entry in files:
        name = entry[0]
        base = os.path.basename(name).lower()
        if base in ('.ds_store', 'thumbs.db'):
            continue
        if name.endswith(('.dll', '.pyd')) and base in _seen:
            continue
        _seen.add(base)
        out.append(entry)
    return out

a.datas = _dedupe(a.datas)
a.binaries = _dedupe(a.binaries)

pyz = PYZ(a.pure)

# UPX (F-14)
UPX_DIR = str(PROJECT_DIR / "tools" / "upx") \
    if (PROJECT_DIR / "tools" / "upx" / "upx.exe").exists() else None
UPX_EXCLUDES = [
    "vcruntime140.dll", "python311.dll",
    "tcl86t.dll", "tk86t.dll", "_tkinter.pyd",
]

exe = EXE(
    pyz, a.scripts, a.binaries, a.datas, [],
    name=f"{APP_NAME}-{VERSION}",
    debug=False,
    bootloader_ignore_signals=False,
    strip=True,                # F-18
    upx=bool(UPX_DIR),
    upx_dir=UPX_DIR,
    upx_exclude=UPX_EXCLUDES,
    runtime_tmpdir=None,
    console=False, windowed=True,
    disable_windowed_traceback=False,
    target_arch=None,
    codesign_identity=None,
    entitlements_file=None,
    icon=str(PROJECT_DIR / "assets" / "icons" / "icon.ico"),
)

7. Questions ouvertes (décision humaine)

  1. ExifTool fallback (F-01) : tolérer la perte du fallback subprocess ? Ou packager une version "deluxe" séparée avec ExifTool pour les utilisateurs avancés ?
  2. AVIF support (F-04) : maintenir AVIF dans la liste des extensions scannées ? (Si non, retirer aussi de EXTENSIONS['image'] dans exif_extractor.py et file_manager.py.)
  3. pi-heif vs pillow-heif (F-03) : confirmer que le projet ne crée jamais de HEIC (= encodage HEVC). Si un bouton "Convertir en HEIC" existe → garder pillow-heif.
  4. --onefile vs --onedir (F-13) : portabilité (1 fichier) prioritaire ou démarrage rapide ?
  5. UPX et antivirus (F-14) : tolérer le risque de faux-positif Windows Defender (heuristique vs UPX) ?
  6. Reprise de l'audit : appliquer les actions par ordre du plan §4 ? Choisir un sous-ensemble ?
