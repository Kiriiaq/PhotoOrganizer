2 🔴 Bloquants avant tout lancement public

2.1 — Générer la vraie SECRET_KEY Pro P0 5 min

La clé actuelle dans validator.py est un placeholder public. Si tu lances avec elle, n'importe qui peut générer ses propres clés. Cette étape DOIT être faite avant le premier build Pro.

  1. Génère une clé aléatoire de 32 bytes : 
    python -c "import secrets; print(secrets.token_hex(32))"
  2. Crée le fichier secret (déjà gitignored, ne sera jamais commit) : 
    notepad src\photoorganizer_pro\license\_secret.py
    Colle dedans : 
    SECRET_KEY = b"a8d2f3e9b1c5d7e4a6b8c0d2e4f6a8b0c2d4e6f8a0b2c4d6e8f0a2b4c6d8e0f2"  # remplace par ta clé hex
SECRET_KEY = bytes.fromhex(SECRET_KEY.decode())
    Adapte avec ta vraie clé. Sauve.
  3. Vérifie que le module l'importe : 
    python -c "from src.photoorganizer_pro.license.validator import SECRET_KEY; print(len(SECRET_KEY), 'bytes')"
    Tu dois voir 32 bytes.
  4. Sauvegarde cette clé dans un password manager. Si tu la perds, toutes les clés générées avant deviennent invalides → chaos pour les acheteurs.
  5. Lance les tests pour vérifier que rien n'est cassé : 
    python -m pytest tests/functional/test_pro_license.py -v
    Les 14 tests doivent passer (avec ta nouvelle clé).

2.3 — Cadre juridique (auto-entreprise, IP, clauses employeur) P0 30 min admin + 1 h vérif

Vendre une licence Pro sans statut légal = facturation impossible + risque administratif. À fermer avant la première vente.

  1. Vérifie ton statut actuel :
    • Si tu as déjà une auto-entreprise (micro-entreprise) ou EURL : OK, vérifie juste que l'activité couvre "édition de logiciel".
    • Si tu n'as rien : crée une auto-entreprise. C'est gratuit et en ligne en 5 minutes : entreprendre.service-public.fr → "Créer une micro-entreprise". Activité = "Édition de logiciels" (code APE 5829C).
  2. Vérifie la clause employeur (si applicable) :
    • Si tu as un emploi salarié : relis ton contrat. Cherche les mots "propriété intellectuelle", "œuvres créées", "clause de cession".
    • Si une clause s'applique aux projets perso → demande une autorisation écrite à ton employeur (mail RH suffit en général) avant de monétiser.
    • Vérifie aussi : as-tu commit depuis un poste/réseau employeur ? Si oui, attention.
  3. Choisis ton nom commercial :
    • Garder le pseudo Kiriiaq pour la visibilité projet.
    • Mais les factures, contrats, déclarations doivent porter ton nom légal Emmanuel Grolleau. Lemon Squeezy demande le nom légal lors du setup.
    • Le LICENSE-PRO existant est déjà rédigé au nom de Kiriiaq (Emmanuel Grolleau), donc OK.
  4. Retire le bundle ExifTool du build (compatibilité license GPL) :

    L'audit a montré qu'ExifTool (Perl, GPL) est bundlé dans assets/tools/ pour 10 Mo. Le fallback subprocess est cassé en pratique (chemin invalide), et bundler du GPL avec une distribution Pro propriétaire est juridiquement ambigu. Action recommandée : appliquer le finding F-01 de l'audit EXE avant le premier build Pro.

    git rm -r assets/tools/
    Puis modifier build.py:178 pour ne plus inclure assets entier, mais seulement assets/icons.

3.4 — Setup Lemon Squeezy + webhook P1 2-3 h

La plateforme de vente. Gère TVA EU + facturation auto. Tu peux laisser la page en mode brouillon et publier silencieusement avant le lancement officiel.

  1. Créer un compte Lemon Squeezy (gratuit) : app.lemonsqueezy.com/register
  2. Setup store :
    • Nom : PhotoOrganizer
    • Slug : photoorganizer (URL deviendra photoorganizer.lemonsqueezy.com)
    • Default currency : EUR
    • VAT : "Yes, collect VAT" (Lemon Squeezy gère, tu n'as rien à faire)
  3. Créer 3 produits dans Products → New :
    ProduitPrixVariantes
    PhotoOrganizer Pro Personal19 € one-timeaucune
    PhotoOrganizer Pro Studio49 € one-timeaucune
    PhotoOrganizer Pro Lifetime99 € one-timeaucune
    Description : reprendre celle de docs/MONETIZATION.md §3.3.
  4. Promo code de lancement : Discounts → Add → code EARLY30, -30 %, expire 30 jours après création.
  5. Webhook pour générer + envoyer la clé :
    • Settings → Webhooks → Add endpoint.
    • URL : https://<ton-domaine-ou-ngrok>/lemonsqueezy/webhook (voir option ci-dessous).
    • Events à cocher : order_created et order_refunded.
  6. Implémenter le handler webhook — 2 options :
    • Option A (recommandée, gratuit) : déployer un script Python ultra-court sur Cloudflare Workers ou Vercel (free tier). Le script reçoit le webhook, génère la clé avec keygen.py, envoie un email avec la clé via Resend ou Brevo (free tier 100 emails/jour).
    • Option B (manuel au début) : Lemon Squeezy envoie l'email d'achat avec un lien de téléchargement. Tu te connectes manuellement, génères la clé : 
      python -m src.photoorganizer_pro.license.keygen --email client@example.com --edition PERS --years 1
      Et tu réponds à l'email du client avec la clé. Tenable pour les 10 premières ventes, à automatiser après.
  7. Page de remerciement (Settings → Checkout → Thank-you page) : "Tu vas recevoir ta clé par email d'ici 1 minute. Si rien après 5 min, écris à manugrolleau48@gmail.com."
  8. Test du flow complet en mode test (Lemon Squeezy a des cartes de test).
  9. Publier la page produit (Status → Published) mais ne pas la promouvoir tant que tu n'as pas testé l'envoi de clé.

3.5 — Publier sur PyPI (core gratuit) P2 2-3 h

Permet l'install pip install photoorganizer. Pas critique mais bon pour la crédibilité dev.

  1. Créer un compte PyPI + activer 2FA (obligatoire depuis 2024) : pypi.org/register
  2. Vérifier que le nom est libre : 
    curl -s https://pypi.org/pypi/photoorganizer/json | head -c 50
    Si tu vois {"error":"Not Found"}, c'est libre. Sinon prends photoorganizer-kiriiaq ou pyphotoorganizer et adapte le name dans pyproject.toml.
  3. Construire les artefacts : 
    python -m pip install --upgrade build twine
python -m build
ls dist/
    Tu dois voir photoorganizer-2.2.0-py3-none-any.whl et photoorganizer-2.2.0.tar.gz.
  4. Tester en local (dans un autre venv) : 
    python -m venv /tmp/test-install
/tmp/test-install/Scripts/Activate.ps1
pip install dist/photoorganizer-2.2.0-py3-none-any.whl
photo-organizer
    L'app doit se lancer.
  5. Upload vers TestPyPI (sandbox) d'abord : 
    twine upload --repository testpypi dist/*
  6. Si OK, upload PyPI prod : 
    twine upload dist/*
  7. Vérifier en live : pypi.org/project/photoorganizer/

Alternative pro : configurer PyPI Trusted Publisher dans GitHub Actions pour upload auto à chaque tag (pas de token API à gérer). Voir leur doc, 15 min de plus.
